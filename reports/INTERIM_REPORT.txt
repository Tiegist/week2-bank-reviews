====================================================================================================
1. DATA COLLECTION METHODOLOGY
====================================================================================================

1.1 Overview
----------------------------------------------------------------------------------------------------

User reviews were collected from the Google Play Store using the google-play-scraper
Python library. The scraping process was designed to collect a minimum of 400 reviews
per bank, targeting a total of 1,200+ reviews across all banks.

1.2 Data Sources
----------------------------------------------------------------------------------------------------

Source: Google Play Store
Collection Method: Automated web scraping using google-play-scraper library
Target Apps:
  • Commercial Bank of Ethiopia (CBE): com.combanketh.mobilebanking
  • Awash Bank: com.sc.awashpay
  • Amharabank: com.amharabank.Aba_mobile_banking

1.3 Data Collected
----------------------------------------------------------------------------------------------------

For each review, the following information was collected:
  • Review text (user feedback)
  • Star rating (1-5)
  • Review date
  • Bank/app name
  • User name (anonymized)
  • Thumbs up count
  • Source identifier

1.4 Collection Statistics
----------------------------------------------------------------------------------------------------

Total Reviews Collected: 967

Reviews by Bank:
  • Commercial Bank of Ethiopia: 400 reviews (41.4%)
  • Awash Bank: 400 reviews (41.4%)
  • Amharabank: 167 reviews (17.3%)

2. DATA PREPROCESSING
====================================================================================================

2.1 Preprocessing Steps
----------------------------------------------------------------------------------------------------

The raw scraped data underwent comprehensive preprocessing to ensure data quality
and consistency:

Step 1: Missing Data Handling
  • Removed rows with missing review text
  • Removed rows with missing ratings
  • Filled missing user names with 'Anonymous'
  • Filled missing optional fields with appropriate defaults

Step 2: Date Normalization
  • Converted all dates to YYYY-MM-DD format
  • Extracted year and month for temporal analysis

Step 3: Text Cleaning
  • Normalized whitespace (removed extra spaces)
  • Removed empty reviews
  • Calculated text length metrics

Step 4: Data Validation
  • Validated ratings (ensured 1-5 range)
  • Removed invalid ratings
  • Verified data consistency
